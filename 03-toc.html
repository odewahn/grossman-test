Activity-Based Learning Ruby

Learn how to program in Ruby and get exposed to computer science principles and techniques along the way.  You’ll get exposed to concepts when they’re needed and ensure you understands those concepts before moving on and building on your knowledge. You’ll start out being an active participant in building your own Blackjack card game by writing, testing, and running code as part of the experience without having to install a Ruby environment.

After completion, you’ll have the skills to solve problems and use algorithms with elegance and inspiration rather than simply plowing ahead with brute force. So when faced with a new problem, you will not just throw up your hands, you will have a methodical approach to solving it.

Key Activities:

  • Talking to the user with Ruby -- the puts command.  We'll use this to prompt the user for the decisions needed during a game of blackjack.
    • puts command
  • Listening to the user.  We have to be able to receive input.  We'll do this with gets.
    • gets command
  • Deciding what to do next.  We need to use an IF-THEN statement to decide if we should keep playing or not.
    • variables
    • assigning a value to a variable
    • if then else
    • if then
  • Dealing Cards:  We have to define a card object and and populate the cards with all their numbers and all their suits.
    • Arrays
    • using objects
    • initializing objects
    • objects, what are they
  • Deciding on HIT or STAND for the dealer.  We now need a DEALER object to decide if we take a HIT or STAND based on the rules of black jack.  
    •  
  • Keeping track of the player's score during a game.  We need a PLAYER object to keep track of all the cards the player has.

 

  • Updating the Scoreboard.  Once a hand of blackjack is over, we need to update how much money the player has or how many games the player has won.  We need a SCOREBOARD object for this. 
  • Storing the results of the game to a file.  If we want to stop playing for a while and keep track of the game status we need a FILE to do this so we have to learn how to open and WRITE to a file.
    • Write to File
  • Reading the existing game file.  To restore status we need to READ from the file we just created.
    • Read from File
